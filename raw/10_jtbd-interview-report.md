## JTBD CARD #01 — [Core] 감사 대응 Job (Audit Readiness)

| 항목 | 내용 |
|---|---|
| **유형** | Core |
| **페르소나** | 박품질 (HBM/EUV 핵심 부품사 품질팀장) |
| **AOS / DOS** | **4.0 / 3.8** → Q1 핵심 혁신 기회 |
| **Situation (When)** | 글로벌 팹 실사관이 현장에 도착하여 특정 로트의 검사 성적서와 소급(Traceability) 데이터를 즉석에서 요구할 때 |
| **Job Statement (I want to)** | 엑셀과 종이 바인더를 뒤지는 대신, **단 1분 안에 완벽한 디지털 증빙 자료**를 찾아내어 실력을 증명하고 싶다 |
| **Desired Outcome (So I can)** | 오딧 중 데이터 누락으로 인한 '부적합' 권고 Zero 달성 및 심사 준비 시간 90% 단축 |
| **🔴 Push** | 엑셀 관리 한계로 인한 데이터 누락 공포; 감사 전후의 극심한 번아웃; *"그 15분이 15년처럼 느껴졌습니다"* |
| **🟢 Pull** | "1분 오딧 대응", "디지털 소급 자동화" 기능의 압도적 편리함 |
| **⚪ Habit** | 10년간 사용해온 종이 성적서·바인더 체계의 물리적 안도감 |
| **🟡 Anxiety** | "클라우드 도입 시 고객사 도면·기밀 데이터가 유출될까 봐 걱정됨" |
| **현재 솔루션** | 대량의 엑셀 시트(수기 입력), 연도별 종이 바인더, 법정 인증 심사원 자문 |
| **Switch Trigger** | 성적서 낱장 하나를 못 찾아 '경고'를 받고 원청 등급이 하락됐을 때 |
| **Switch Barrier** | 수만 건의 과거 종이 데이터 디지털화의 막막함 → **해결: 자동 OCR 마이그레이션 제공** |
| **Customer Job (VP)** | 감사 대응 문서 실시간 소급(Traceability) |
| **Pain (VP)** | 로트 데이터 누락 → 원청 등급 하락 → 매출 직결 위협 |
| **Gain (VP)** | 1분 내 완벽한 디지털 증빙 출력 → 오딧 자신감 회복 |
| **Pain Reliever** | AI 소급 추적 엔진 + 디지털 성적서 자동 생성 |
| **Gain Creator** | Audit-Ready 실시간 대시보드 |
| **MVP 모듈** | **Audit Module** — Phase 1 🥇 |
| **핵심 인용** | *"실력은 100점인데 데이터 찾는 속도가 0점이면, 결국 불신만 남더라고요."* |

### 성과 지표 (Before → After)

| 지표 | Before | After | 변화율 |
|---|---|---|---|
| 오딧 중 데이터 탐색 시간 | 평균 15분 | **1분 이내** | ↓ 93% |
| 부적합(NC) 발생 건수 | 연 2~3건 | **Zero** | ↓ 100% |
| 심사 준비 행정 시간 | 보름 (전사 집중) | **단축** | ↓ 90% |
